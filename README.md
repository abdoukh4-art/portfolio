# Portfolio — Abderrahmane Khellou

Personal portfolio: first-year software engineering (GL) student at ENSIAS, Rabat.
Built with Next.js, TypeScript and Tailwind CSS. English by default, with a French toggle.

## Run it locally

```bash
npm run dev
```

Then open [http://localhost:3000](http://localhost:3000).

## Edit the content

**All text on the site lives in one file: [`lib/content.ts`](lib/content.ts).**
It has two objects, `en` and `fr` — keep them in sync (same keys) when you edit.

Common edits:

- **Add a project** — add an entry to `projects.items` in both `en` and `fr`.
  Set `link` to the GitHub repo URL once it's public, and `active: true` if it's
  in progress (shows the pulsing dot).
- **Update the "Now" log** — add a line to `now.entries`. It's meant to be
  updated by hand whenever what you're learning changes.
- **Move a skill between groups** — the three groups in `skills.groups` are
  "comfortable with", "actively exploring", "next on the list". Promote things
  as you learn them.

Design tokens (colors, fonts) are in [`app/globals.css`](app/globals.css).
Layout and components are in [`components/site.tsx`](components/site.tsx).

## The "Learn how to" system

`/learn` hosts active lessons generated by **prof** (`.claude/agents/prof.md`),
a teaching subagent with strict laws: one concept per file, always anchored in
real project code, always ending with a test that can make the student fail.

- Lesson files live in `apprentissage/<topic-slug>/concept-<name>.md`.
- Topics (order of lessons included) are declared in [`lib/learn.ts`](lib/learn.ts);
  their bilingual cards live in the `learn` key of [`lib/content.ts`](lib/content.ts).
- Each lesson page renders writing zones (cold answers, test answers, journal)
  that persist in the reader's browser only (localStorage).
- To add a topic: invoke prof on a project, then register the topic in the two
  files above. Restart the dev server after adding a topic (static params are
  cached).
- Duplicated concepts are never re-explained: lessons link to the existing
  lesson (`/learn/<topic>#<slug>`) and add only case-specific details.

## Deploy

```bash
npm run build
```

The site is fully static. Easiest deploy: push to GitHub and import the repo on
[Vercel](https://vercel.com/new) — free, with a `*.vercel.app` URL.
